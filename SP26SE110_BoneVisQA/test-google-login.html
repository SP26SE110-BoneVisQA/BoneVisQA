Google sign-in test (API only)

This page calls POST /api/auths/google-login with the Google ID token. Use the same OAuth Web client ID as in your backend Google:ClientId (User Secrets / environment).

Google Cloud Console → Credentials → your OAuth 2.0 Client ID → Authorized JavaScript origins must include your site origin exactly (e.g. http://localhost:3000, your Vercel URL). If you see “The given origin is not allowed for the given client ID”, add the missing origin here.

CORS: That origin must also appear under Cors:AllowedOrigins in API appsettings / secrets so the browser can call the API with credentials if needed.

Optional URL params: ?api=http://localhost:5046&clientId=YOUR_WEB_CLIENT_ID.apps.googleusercontent.com
